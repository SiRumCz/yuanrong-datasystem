== Q1: What problem does this bounded context solve, and for whom?

=== Q1.1: Product identity — what is this bounded context?
[ANSWERED]
Evidence: README_EN.md:34, README_EN.md:36-45, VERSION:1
openYuanrong DataSystem is a distributed caching system that pools HBM/DRAM/SSD
across a compute cluster to build a near-compute multi-level cache, exposing
object, KV, stream, and heterogeneous-(NPU)-object semantics. It is the data
system repository of the three-repository openYuanrong project; current version
is 0.8.2 (VERSION:1).

==== Q1.2.1: What software actors call the system?

The direct users are application processes that link an SDK: C++ via the

=== Q2.2: Use-case catalog
==== Q2.2.PUC.KvPutGet: KV set/get/delete a value
===== Q2.2.PUC.KvPutGet.Trigger
[ANSWERED]
Evidence: include/datasystem/kv_client.h:133 (Set), kv_client.h:258 (Get), kv_client.h:354 (Del)
An application calls KVClient::Set(key,val,param) to store a value, Get(key,val)
to read it, Del(key) to remove it; MSet/MGet (kv_client.h:216, :312) batch them.
===== Q2.2.PUC.KvPutGet.MainSuccess
[ANSWERED]
Evidence: include/datasystem/kv_client.h:133, kv_client.h:427 (Exist), kv_client.h:441 (Expire)
Value is written to shared memory (zero-copy), optionally with TTL via Expire;
Exist checks presence. For ExistenceOpt::NX, an existing key returns K_OK and is
left unchanged (kv_client.h:130-133).
===== Q2.2.PUC.KvPutGet.Durability
[ANSWERED]
Evidence: include/datasystem/object/object_enum.h:30-35 (WriteMode), src/datasystem/common/l2cache/persistence_api.h:32-100
Durability is governed by WriteMode: NONE_L2_CACHE (cache only), WRITE_THROUGH
(sync to L2), WRITE_BACK (async to L2), plus evictable variants; persistence is
performed through PersistenceApi::Save/Get/Delete.

==== Q2.2.PUC.ObjectRefCounted: Create/publish/get a reference-counted object
===== Q2.2.PUC.ObjectRefCounted.MainSuccess
[ANSWERED]
Evidence: include/datasystem/object_client.h:84 (GIncreaseRef), object_client.h:130 (Create), include/datasystem/object/buffer.h:67 (MemoryCopy), buffer.h:79 (Publish), object_client.h:158 (Get), object_client.h:99 (GDecreaseRef)
Client increments global ref, Create returns a shared-memory Buffer,
MemoryCopy writes data, Publish/Seal makes it immutable and visible, Get returns
buffers; the object lives while global ref > 0 and is reclaimable at 0.
===== Q2.2.PUC.ObjectRefCounted.RefCounting
[ANSWERED]
Evidence: include/datasystem/object_client.h:84, object_client.h:99, object_client.h:117 (QueryGlobalRefNum)
Lifecycle is reference-counted across the cluster via GIncreaseRef/GDecreaseRef
with QueryGlobalRefNum to inspect the current count.

==== Q2.2.PUC.StreamPubSub: Produce/consume stream elements
===== Q2.2.PUC.StreamPubSub.MainSuccess
[ANSWERED]
Evidence: include/datasystem/stream_client.h:80 (CreateProducer), include/datasystem/stream/producer.h:59 (Send), include/datasystem/stream_client.h:97 (Subscribe), include/datasystem/stream/consumer.h:59 (Receive), consumer.h:89 (Ack)
A producer is created on a named stream and Sends elements; a consumer
Subscribes, Receives up to expectNum elements within a timeout, and Acks them.
DeleteStream (stream_client.h:117) tears the stream down.

==== Q2.2.PUC.HeteroNpu: Move data between NPU device memory and host
===== Q2.2.PUC.HeteroNpu.MainSuccess
[ANSWERED]
Evidence: include/datasystem/hetero_client.h:72 (MGetH2D), hetero_client.h:83 (MSetD2H), hetero_client.h:153 (DevMGet), hetero_client.h:165 (DevMSet), hetero_client.h:182 (DevDelete)
Hetero client abstracts NPU HBM as keyed objects: MSetD2H/MGetH2D move between
